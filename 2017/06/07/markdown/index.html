MarkDown学习笔记

发表于 2017-06-07 更新于 2020-06-04 分类于 学习笔记 阅读次数：

前言

第一次触电大概是半年前，那时候项目需要使用到Github，然后就被那个熟悉的不能熟悉README.md深深吸引了，感觉这个格式非常适用于生产模式。在这个“无主题，不颜值”的时代，Markdown让作者们只需要关注自己的生产内容，无论是程序猿们还是媒体人，都可以在这种文档实现自己想要的效果。

优势

就切身体验而言，它

-跨平台
-对程序猿非常友好
